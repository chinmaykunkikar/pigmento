## Vision

### 10x Check
Pika becomes two things with one index. For humans: the asset manager for any codebase, open it on a repo and see the design system that actually exists (images, colors, typography as peer kinds), rendered minimal, finished, shareable. For agents: the design ground-truth layer, the MCP server every harness mounts before touching UI code, answering "what color should I write here?", "does an icon like this exist?", "did I just introduce drift?" from the codebase itself. Every named competitor (Figma MCP, design-system MCP servers) serves an authored system; pika derives the system from code, so the 95% of repos with no formal design system get one for free. Distribution inverts: instead of convincing humans to install a tool, pika rides agent registries and harness loadouts.

### Audience (revised from June)
Frontend developers (direct users, npx + MCP), PMs and designers (consumers of shared artifacts: the poster; the app when a dev opens it for them), coding agents (first-class API consumers).

5. **Agent surface bundle**: `pika mcp` (stdio) exposing 5 read tools, contracts: get_palette (input: none; output: brand palette + per-color usage counts, colors kind); resolve_token_for_value (input: any CSS color string, normalized via the engine's OKLCH path; output: nearest existing token + ΔE distance, or "no token within threshold", threshold = the engine's ΔE2000 < 6 clustering constant); find_similar_asset (input: image file path, optional top-N default 10 max 50; output: indexed assets with similarity scores, images kind, reuses the CLIP match pipeline); get_typography_scale (input: none; output: font families, size scale, weights with usage counts); list_drift (input: optional kind + optional top-N, default 10; output: drift findings with file:line and suggested token). Plus `pika check --json` advisory (agent self-correction, no exit-code gating), `pika context` (compact markdown digest: brand palette, type scale, tokenization conventions, canonical asset dirs; stable section order for prompt caching), and `pika init` (idempotent marker-delimited append of agent guidance; writes its block to each of CLAUDE.md and AGENTS.md that exists, creates AGENTS.md when neither exists; re-run replaces its own block, never duplicates). CLI cold-start policy: `check --json` and `context` follow D4.3 (synchronous auto-index when no index exists); `init` needs no index and never triggers one.
6. **Packaging**: npx two-minute path per the packaging step of the companion design doc (`~/.gstack/projects/chinmaykunkikar-pika/chinmay-preview-backdrop-toggle-design-20260610-035546.md`), under the NEW product name (see naming workstream).
7. **Launch content kit**: demo video/GIF, README rewrite around the asset-manager frame, launch post draft, posters from 3 famous OSS repos, MCP registry + awesome-list submissions.
8. **Delight pack** (riders on reskin/engine tasks): click-to-copy color with toast; poster swatch hover (usage count + top file); CLI ascii palette after index; `/` unified search (S-sized, not a micro-touch: searches already-indexed names and values via the existing FTS table plus the new style tables; no new search index); typography clean-verdict state.

## Implementation decisions resolved now (0E)
- **npm identity (D4.1)**: new product name, workstream above. One-way door at publish.
- **Routing (D4.2)**: kind-level App Router routes; cluster-level deep links deferred, layerable without rework.
- **MCP cold start (D4.3)**: USER OVERRIDE of recommendation: first tool call on an unindexed repo triggers a synchronous full index. Recorded risk: large repos exceed harness tool timeouts, and progress notifications do not extend most harnesses' own timeout budgets. Implementation notes: emit MCP progress notifications for harnesses that honor them; document expected first-call latency and retry guidance in every tool description; on client cancellation, abort cleanly (a cancelled MCP request's response is discarded, so no partial answer is returned). Cancellation is NOT stage failure under the inherited OV-1(b) purge rule: completed stages persist their rows and lastRun stamps; the interrupted stage purges and re-runs on retry (per-stage resume only, no mid-stage checkpointing). Escalation trigger: any dogfood cold start over 60 seconds promotes the deferred background-auto-index fallback into the next release.
- **Reskin depth (D4.4)**: full design-language rev in one pass alongside the IA work; no staged half-reskin.
- **Advisory check semantics**: "introduced" = working tree vs the live last-indexed state (the index itself is the advisory baseline; separate baseline persistence is only needed for the deferred CI gate). Diff-scoped when a ref range is provided, meaning findings are filtered to files changed in that range (no extraction at historical refs); whole-repo otherwise. Output is JSON findings, never exit-code gating in this arc.
- **Poster rendering**: server-side compose over indexed data (sharp already a dependency), OG-card dimensions, downloadable from the overview; also emitted by CLI for the launch-kit famous-repo runs.
