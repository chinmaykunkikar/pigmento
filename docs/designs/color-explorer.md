## Vision

### 10x Check
Not "a color tab" — pika opens on any repo and renders the design system that *actually exists* (colors, type, spacing, radii, shadows), scored against the system the repo *claims* in its tokens, with an agent standing by to close the gap. Dembrandt does this for websites from the outside; nobody does it from inside the codebase, where the fix lives. The session eureka (independently reached by Codex): every existing tool is a gate or a report; pika's index → cluster → canonical → agent-PR loop makes drift a cleanup problem with an executed fix.

### Platonic Ideal
Point pika at a five-year-old production app. Two minutes later: a poster of the app's true visual identity — six brand colors radiating usage, a faded ring of forty accidental variants around them, each carrying the commit that introduced it and a one-click "collapse into canonical" that becomes a PR. An X-ray, not a report card. Nothing nags; everything is fixable.

## Implementation decisions resolved now (0E temporal interrogation)
- **Archaeology mechanism (E6):** first-introduction of the literal via per-file `git log -S<token>` taking the oldest commit — the question being answered is "when did drift enter." Per-file scoping + the existing 500ms execFile timeout pattern are the perf guards; archaeology is per (file, token) and line-agnostic, so null-line rows work identically; "unknown" is reserved for timeout, shallow clone, or no matching commit.
- **Typography normalization baseline (E1):** normalize length units to px at a 16px root assumption for comparison/clustering; store the raw unit. **Unitless line-height is exempt** — stored raw and compared as a ratio. Mixed-unit equivalence (13px vs 0.8125rem) is a *finding*, not noise.
- **Typography clustering semantics (E1), v1 rules:** exact-group on (property, normalized value); near-miss for font sizes = values within 1px of a more-popular neighbor (13 vs 13.5); font-family near-miss = same first family, differing fallback tails. No ΔE analog pretensions; calibrate in dogfood like the color threshold.
- **Tailwind named utilities (both kinds):** treated as already-tokenized (count toward coverage, not drift) in v1, detected by class-name pattern heuristics (`bg-*`, `text-*`, `font-*` against the known default scale) — NOT config resolution, which stays descoped. If the heuristic feels too loose in dogfood, narrow the v1 coverage denominator to CSS + arbitrary-value surfaces and say so in the UI.
- **Replace-with-var tie-break (E2):** when multiple vars define the same canonical color, the most-referenced var wins; ties surface as a choice in the plan builder rather than guessing.
- **Coverage-score denominator (E2):** usages, not distinct values — 200 tokenized usages + 2 literals should score high.
- **Hour-1 implementer entry point:** spike script first (design doc step 1), schema second; the spike informs contextDetail semantics before freeze.

## Deep-review decisions (11-section CEO review, 2026-06-10)

| S3 | Security | `open` server bind address stays at server default — explicitly accepted risk (user decision; loopback-only was recommended and declined). Constraint kept: archaeology git calls use the execFile argument-array pattern, never shell interpolation. |
| S6 | Tests | Testing DEFERRED (user decision; vitest + fixture engine tests was recommended and declined). The zero-FP success criterion is defended by manual dogfood verification. Tracked in TODOS. |
| S7 | Performance | Archaeology runs as a BACKGROUND BACK-FILL stage after the index run completes: UI opens immediately, history rows appear as the trailing stage lands (no spinner; trailing progress visible via S2 counts). Pairwise ΔE clustering stays at index time; bucket by OKLCH lightness only if dogfood shows pain. Indexes: (kind, normalizedColor) and (kind, sourceId) on style_usages. |
| S9 | Deployment | Plan-format v1→v2: TOLERANT UNION PARSER — schema reads both versions, normalizes v1 to v2 in memory; no data loss, no migration step. |
| S11 | Design/UX | Clean-repo EMPTY state is a designed verdict screen: coverage score, "0 suspicious near-misses across N usages," the discovered brand palette, pointer to one-offs. Palette grid virtualizes per the >200-row house rule. |
